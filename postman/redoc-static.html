Diplomayn API (1.0.0)

Download OpenAPI specification:

Complete collection for the dip Express backend.

Variables

  • baseUrl — e.g. http://localhost:5050 (no trailing slash).
  • token — paste accessToken from Register or Login into this variable for protected routes.
  • refreshToken — paste from Register/Login for the Refresh request body field mapping.
  • userId — target user id for GET/PATCH /api/users/:id.
  • conversationId — id from chat list or create response.

Notes

  • Register requires teacher_ids (array of existing user ids with role TICHER).
  • Refresh body field is refresh_token (snake_case); the collection variable refreshToken is referenced as {"refresh_token":"{{refreshToken}}"}.
  • Admin routes need a user with role ADMIN.
  • POST .../messages/stream returns SSE (text/event-stream); Postman may buffer or behave differently than curl/browser.

Auth

Registration, JWT login/refresh, and self-service profile.

Register student

Creates a new STUDENT user, links them to teachers, hashes password, and returns JWTs.

Body fields

  • email — unique contact email.
  • username — unique, 3–30 alphanumeric chars.
  • password — min 6 characters (stored hashed).
  • first_name, last_name — display name.
  • teacher_ids — required non-empty array of user ids whose role is TICHER; invalid ids return validation error.
  • gender — optional enum: MALE, FEMALE, or UNKNOWN.

object

Chat

OpenAI-backed fitness assistant. Requires valid OpenAI config on server.

Create conversation

Starts a conversation. If first_message is omitted, returns empty messages and no AI call. If provided, server calls OpenAI, persists user+assistant messages, may auto-set title from message prefix.

Body fields

  • title — optional string up to 120 chars (blank ok).
  • first_message — optional first user turn (1–4000 chars).

Diet

Ranked diet recommendations from static dataset using BMI/age/gender.

Random / top diet recommendations

Computes BMI from UserSettings.weight_kg and height_sm (both must be positive) and scores catalog entries. Returns top matches with closeness_percentage plus success, bmi, and normalized age from profile. If weight/height missing, API responds with validation error (HTTP 422 from error handler).
